Introduction

Since the start of the MoSKito Project we were using following definiton of MoSKito:

MoSKito is a multi-purpose, non-invasive, interval based monitoring system kit for collection, storage and instant analysis of applicationâ€™s performance and behavior data.
Apparently this definition won't help you much. Instead we are going to concentrate on what can it be used for? and get back to the features later.
MoSKito concentrates on three aspects of your application:

  1. Health Monitoring - Is my Application doing OK? Do I have to worry about next night?
  2. Performance Monitoring - Why is my Application slow? Will my Application become slow eventually?
  3. Architecture Monitoring - Does my Application work as intended? How does it work actually?
